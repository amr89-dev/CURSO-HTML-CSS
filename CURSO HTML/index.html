Etiquetas semanticas de maquetacion

  • <div> es una etiqueta contenedora que no tiene semantica
  • < header> cabecera de un sitio web o seccion
  • <main > Define la seccion principal del documento, solo puede haber una por documento
  • < footer> pie de pagina de un sitio web o una seccion
  • <nav > conjunto de enlaces que formen un menu de navegacion
  • < article> representa una seccion de autocontenido(que por si sola se explica)
  • < aside> representa un contenido complementario osecundario (zonas de publicidad)
  • <section > representa una seccion de contenido generico
  • < address> representa una direcion fisica "calle tal" o tambien datos de contacto

Atributos de los Inputs y Formularios

Para el titulo de los formularios usamos <legend>. El elemento mas basico de los formularios son los <inputs> el cual nos dibuja una caja en el documento. Hay varios tipos de inputs como los checkbox, radio, date, color, submit para enviar formularios y reset para restablecerlos. Otro tipo de input es del tipo file que nos sirve para seleccionar un archivo o varios pasandole el atributo multiple, este inputo se puede completar con la etiqueta <progress> que muestra un barra de carga. Tambien hay inputs de tipo hidden "ocultos" que se pueden usar para enviar datos con con el nombre de la variable en name y el valor en value. Hay inputs de tipo email, el cual compruba que lo que introduzcamos tenga un @ o en teclados tactiles agrega los botones de direccion de correo. El input tipo number verifica los datos de entrada como numeros. Y el de tel que sea un telefono. El de password oculta lo que escriba el usuario.









25% - Nombre del archivo.txt






Existe otro tipo de input llamado <textarea> que es una caja de comentarios

Otro tipo de input es el <select> que es una lista de seleccion, dentro debe llevar las etiquetas <option> para seleccionar y el atriburo multiple si queremos permitir una seleccion multiple


Tambien podemos declarar cualquier input de solo lectura con el atributo readonly y disabled. Si queremos poner un texto a modo de indicacion usamos el atributo placeholder


La etiqueta para los formularios es <form> y es una etiqueta de tipo bloque, a este etiqueta se le pasas el atributo action que es el que determina a donde se va a enviar el formulario. Hay que nombrar los inputs con el atributo name para que la url a la que enviemos el formulario detecte cada uno de los inputs. Es importante destacar que la url que se enviar lo que va despues del signo de interrogacion ? es el nombre de la variable y lo que va despues del signo de igualdad = es el valor de dicha variable. Siempre hay que poner el atributo method que es el que indica la forma en que se va a enviar el formulario, los valores de metodo pueden ser GET y POST, GET lo envia a traves de la URL y POST lo envia de manera oculta. Si no ponemos el atributo action, el formulario se autoprocesa. Una etiquete que ayuda mucho es la de <label> pues la podemos usar para identificar los elementos de una frimulario. El atributo required obliga a llenar el formulario. El atributo pattern usa expresiones regulares para validar una entrada. El traibuto tittle le indica una instruccion sobre un input al usuario

Metaetiquetas para SEO y Móviles

Para el SEO dos de las etiquetas mas importantes son <title> y la etiqueta <meta> pues son las que usan los motores de busqueda. Tambien podemos usar la etiqueta <link> con el atributo rel canonical y en href la URL canonica. Otras etiquetas que no son de SEO pero si son necesarias para mejorar la experiencia de la pagina es favicon que preferiblemente debe ser en formato PNG, debe ser cuadrado y se manda allamar tambien con una etiqueta link y el rel le ponemos icon, y para los dispositivos moviles el rel lo ponemos en apple-touch-icon. Hay otra meta etiqueta que nos permite cambiar el color de la barra de direcciones, es una meta etiqueta a la que le pasamos el tel theme-color y el content en color hexadecimal
